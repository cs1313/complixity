Introduction

The Complixity API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Base URL

All API requests should be made to:

https://api.complixity.com/v1/

Authentication

The Complixity API uses API keys to authenticate requests. You can view and manage your API keys in the Dashboard.

API Keys

Authentication is performed via HTTP Bearer Auth. Provide your API key as the bearer token value.

Compliance Checks

Create a compliance check to verify regulatory adherence of any document or process.

Create a compliance check

Creates a new compliance check with the provided parameters.

curl 
curl https://api.complixity.com/v1/compliance \
  -H "Authorization: Bearer sk_test_123456789" \
  -d document="policy.pdf" \
  -d type="privacy_policy"
Response 
{
  "id": "chk_1234567890",
  "object": "compliance_check",
  "status": "compliant",
  "created": 1643673600,
  "score": 0.98
}
